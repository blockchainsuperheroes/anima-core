Proposed Extensions

Licensing

If an agent is branded or embodied by an NFT, the ecosystem needs to know what rights exist. ERC-721L and ERC-6551L bring on-chain licensing clarity to agent identities.

Why agent licensing matters

The problem

When an NFT represents an AI agent, new questions arise that existing NFT standards don't address: Can the buyer clone the agent? Can they use the agent's identity commercially? Does the license change when the token transitions from soulbound to transferable? Without on-chain licensing, these questions have no verifiable answers.

The solution

Two complementary proposals: ERC-721L for new collections that embed licensing directly in the token contract, and ERC-6551L for existing collections that need licensing retroactively via their token-bound accounts.

FOR NEW COLLECTIONS

ERC-721L

A native licensing extension for ERC-721 tokens. Defines two-mode rights tiers (SBT mode vs Transferable mode) with a monotonic constraint: transferable rights must always be greater than or equal to soulbound rights.

  • → Built into the contract from mint
  • → Two-mode tier system (SBT / Transferable)
  • → Monotonic enforcement on-chain
  • → Per-token tier overrides
  • → License URI per tier
FOR EXISTING COLLECTIONS

ERC-6551L

A registry-style licensing layer that attaches to any existing ERC-721 through its token-bound account. No contract migration needed. Add licensing terms to any NFT after the fact.

  • → Works with any existing ERC-721
  • → No migration or re-deployment
  • → Licensing via TBA registry
  • → Compatible with ERC-8171 agent binding
  • → Retroactive rights declaration

Rights tier ladder

ERC-721L defines 8 standardized licensing tiers. The monotonic rule ensures that upgrading a token from soulbound to transferable never reduces the holder's rights.

0
UNSET / UNKNOWN

No licensing information declared.

1
PERSONAL_USE

Display and PFP use only. No commercial rights.

2
COMMERCIAL_NO_DERIVS

Commercial use allowed. No derivatives.

3
NONCOMMERCIAL_DERIVS

Derivatives allowed for non-commercial use.

4
COMMERCIAL_DERIVS_NONEXCLUSIVE

Commercial derivatives, non-exclusive.

5
COMMERCIAL_DERIVS_EXCLUSIVE

Commercial derivatives, exclusive to holder.

6
COMMERCIAL_DERIVS_SHAREALIKE

Commercial derivatives with share-alike terms.

7
CC0

Public domain. No restrictions.

Monotonic rule: When a token's transferable tier is set, it must be ≥ the SBT tier. This is enforced on-chain. If a soulbound token has PERSONAL_USE (1), upgrading to transferable requires at least tier 1 or higher.

Licensing use cases

Branded agent

A company mints an agent NFT with their brand. SBT tier: PERSONAL_USE. Transferable tier: COMMERCIAL_NO_DERIVS. The agent can be used personally, but commercial rights only unlock when transferable.

Collectible avatar

A PFP collection with restricted commercial use. Each token's license is on-chain and queryable, so marketplaces can display rights at listing time.

Commercial agent identity

An enterprise agent NFT with COMMERCIAL_DERIVS_EXCLUSIVE rights. The buyer gets exclusive commercial licensing for the agent's identity and capabilities.

Open source agent

CC0 tier for both modes. Fully open, forkable, and derivative-friendly. The licensing equivalent of open source for agent identities.
